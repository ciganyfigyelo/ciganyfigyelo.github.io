HTML meta tags explained

23 November, 2024

What are them? What is their purpose and use? In this article you'll get the answers with examples.

What is a HTML meta tag?

The <meta> tag defines metadata about an HTML document. Metadata is data (information) about data.

<meta> tags always go inside the <head> element, and are typically used to specify character set, page description, keywords, author of the document, and viewport settings.

Metadata will not be displayed on the page, but is machine parsable.

Metadata is used by browsers (how to display content or reload page), search engines (keywords), and other web services.

Basic meta tags

1. Charset

Specifies the character encoding for the HTML document.

<meta charset="utf-8">

2. Base

Specifies the default directory.

<base href="https://www.mysite.hu/">

3. Viewport

Configures the viewport properties, enabling responsive design on different devices.

<meta name="viewport" content="width=device-width, initial-scale=1.0">

4. Description

Provides a brief description of the document. This is often used by search engines.

<meta name="description" content="This is a webpage about HTML meta tags.">

5. Keywords

Specifies a list of comma-separated keywords relevant to the document.

<meta name="keywords" content="HTML, meta tags, web development">

6. Author

Indicates the author of the document in the code.

<meta name="author" content="Kevin K.">

7. Robots

Informs search engines whether to index and follow the links within the document.

Good to know! The listed "args" are in "negative" form, search engines will not index the pages, images and links, also not archive or show a snippet about the page in the search results.

<meta name="robots" content="noindex, nofollow, noimageindex, nosnippet, noarchive">

8. Theme color

Sets the theme color for most browsers. It changes the look of the browser's UI elements when the site is added to the home screen on mobile.

<meta name="theme-color" content="#000">

9. Apple mobile web app

Specifies whether a web application should run in full-screen mode when launched on an iOS device.

<meta name="apple-mobile-web-app-capable" content="yes">

10. Apple mobile web app status bar

Sets the style of the status bar for a web app on iOS devices.

<meta name="apple-mobile-web-app-status-bar-style" content="black-translucent">

11. Format detection

Prevents automatic detection and formatting of possible phone numbers in a webpage.

<meta name="format-detection" content="telephone=no">

12. Referrer

Specifies the referrer policy for links on a webpage.

<meta name="referrer" content="no-referrer">

13. Generator

Indicates the software or tool used to create the webpage.

<meta name="generator" content="WordPress">

Information/value meta tags

1. Cache controller

Directs browsers not to cache the content of a web page.

<meta http-equiv="cache-control" content="no-cache, no-store, must-revalidate">

2. Content language

Specifies the language of the content on the webpage.

<meta http-equiv="content-language" content="en">

3. Content Security Policy (CSP)

Helps prevent cross-site scripting (XSS) and other code injection attacks.

Learn facts how cross-site scripting works.

<meta http-equiv="Content-Security-Policy" content="default-src 'self'">

4. Expirity

Sets an expiration date in the past, essentially telling the browser the page has expired.

<meta http-equiv="expires" content="0">

5. MIME type options

Prevents browsers from interpreting files as a different MIME type.

<meta http-equiv="X-Content-Type-Options" content="nosniff">

6. Pragma

Provides instructions to the browser not to store a cached copy of the web page.

<meta http-equiv="pragma" content="no-cache">

7. Refresh

Redirects the browser to another URL after a specified time interval (in seconds).

<meta http-equiv="refresh" content="5;url=<https://www.mysite.hu/redirected/>">

8. Strict transport security

Enforces the use of HTTPS for the website.

<meta http-equiv="Strict-Transport-Security" content="max-age=31536000; includeSubDomains">

9. X-Frame options

Prevents a webpage from being displayed in a frame or iframe.

Good to know! Major browsers throw an error and not executing the tag. See below.

<meta http-equiv="X-Frame-Options" content="deny">

10. X-UA-Compatible

Instructs Internet Explorer to use its latest and best version to render the web page.

<meta http-equiv="X-UA-Compatible" content="IE=edge">

Open Graph meta tags

1. Open Graph title

Specifies the title of the Open Graph object (for social media sharing).

<meta property="og:title" content="Mysite">

2. Open Graph description

Provides a description of the Open Graph object (for social media sharing.)

<meta property="og:description" content="This is a sample page for Open Graph testing.">

3. Open Graph image

Specifies the URL of the image for the Open Graph object (for social media sharing).

<meta property="og:image" content="/images/socialshare.jpg">

4. Open Graph URL

Specifies the URL of the Open Graph object (your webpage) for social media sharing.

<property="og:url" content="https://mysite.hu/example-url/">

Twitter/X profile tags

1. Twitter URL

Specifies the URL of the Twitter card associated with the shared content.

<meta name="twitter:url" content="https://mysite.hu/twitter/">

2. Twitter card

Specifies the card type for the Twitter card metadata.

<meta name="twitter:card" content="summary">

3. Twitter site

Associates a Twitter username with the website for Twitter cards.

<meta name="twitter:site" content="@example">

4. Twitter title

Sets the title for the Twitter card.

<meta name="twitter:title" content="Sample Twitter Card">

5. Twitter description

Provides a description for the Twitter card.

<meta name="twitter:description" content="This is a sample Twitter card description.">

6. Twitter image

Specifies the URL of the image for the Twitter card.

<meta name="twitter:image" content="https://mysite.hu/images/
twitter/socailshare.jpg">
